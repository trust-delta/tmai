# ファイルブラウザ

WebUIからディレクトリの閲覧とファイルの表示が可能です。

## 概要

ファイルブラウザでは、プロジェクトディレクトリの探索、ファイルの読み取り、設定ファイルやドキュメントファイルの編集ができます。

## 機能

### ディレクトリブラウジング

登録済みプロジェクトパスからファイルシステムを閲覧できます:

- **ディレクトリ一覧** -- ファイルとサブディレクトリを表示
- **Gitインジケーター** -- Gitリポジトリであるディレクトリにマーク表示
- **ナビゲーション** -- ディレクトリをクリックしてより深く探索

### ファイル読み取り

ファイルをクリックすると内容が表示されます。1MBまでのファイルが読み取り可能です。内容はプレーンテキストとして表示されます。

### ファイル編集

対応する拡張子のファイルは編集可能です:

- `.md` (Markdown)
- `.json` (JSON)
- `.toml` (TOML)
- `.txt` (プレーンテキスト)
- `.yaml` / `.yml` (YAML)

既存のファイルのみ編集可能です -- APIによる新規ファイル作成はサポートされていません。

## プロジェクト管理

設定パネルからプロジェクトディレクトリの登録と管理ができます:

1. ステータスバーの設定ボタン（⚙）をクリック
2. **プロジェクト追加** -- 登録するディレクトリパスを入力
3. **プロジェクト削除** -- 登録済みプロジェクトを削除

登録されたプロジェクトはサイドバーに表示され、ブランチグラフの可視化が有効になります。

## APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/directories` | ディレクトリ内容一覧（クエリ: `path`） |
| GET | `/api/files/read` | ファイル内容読み取り（クエリ: `path`、最大1MB） |
| POST | `/api/files/write` | ファイル内容書き込み（制限付き拡張子） |
| GET | `/api/projects` | 登録済みプロジェクト一覧 |
| POST | `/api/projects` | プロジェクトを追加 |
| POST | `/api/projects/remove` | プロジェクトを削除 |

## セキュリティ

- **パス検証** -- ディレクトリトラバーサルを防止
- **書き込み制限** -- 設定/ドキュメント系ファイルタイプのみ書き込み可能
- **既存ファイルのみ** -- APIによる新規ファイル作成は不可
- **サイズ制限** -- ファイル読み取りは最大1MB
