# GitHub連携

プルリクエスト、CIチェック、issueをtmaiダッシュボード上で直接監視できます。

## 前提条件

- [GitHub CLI (`gh`)](https://cli.github.com/) がインストール済みで認証済みであること
- プロジェクトがtmaiに登録済みであること（設定パネルから）

## プルリクエスト

PRステータスはブランチグラフのブランチ先端に表示されます:

<!-- screenshot: github-prs.png -->

### PRバッジ

オープンPRを持つ各ブランチには以下が表示されます:

- **タイトル** -- PRタイトル
- **ステータス** -- ドラフト、オープン、マージ済み、またはクローズ
- **レビュー判定** -- 承認済み、変更要求、またはレビュー必要
- **レビュー数** -- 提出されたレビューの数
- **コメント数** -- PRコメントの数

### レビューステータスの色分け

| ステータス | 色 |
|------------|------|
| 承認済み | 緑 |
| 変更要求 | 赤 |
| レビュー必要 | 黄 |
| ドラフト | グレー |

## CIチェック

GitHub ActionsからのCI/CDパイプラインステータス:

<!-- screenshot: github-checks.png -->

### チェック表示

ブランチグラフでブランチを選択し、アクションパネルのCIチェックセクションを展開します:

- **ロールアップステータス** -- 全体の成功/失敗/保留中
- **個別チェック** -- 名前とステータス付きの各ワークフロー実行
- **ステータスアイコン** -- 成功（✓）、失敗（✗）、保留中（◌）、スキップ（⊘）

### チェックステータス

| ステータス | 説明 |
|------------|------|
| `success` | チェック成功 |
| `failure` | チェック失敗 |
| `pending` | チェック実行中 |
| `neutral` | 結論なし |
| `skipped` | チェックスキップ |

## Issue

issueはブランチに自動的にリンクされます:

### 自動リンク

tmaiは以下からissue番号を抽出します:

- **ブランチ名** -- `fix/issue-42`、`feat/gh-123`、`42-fix-bug`
- **PRタイトル** -- `Fix #42: resolve timeout`

リンクされたissueの表示内容:

- issue番号とタイトル
- 色付きラベル
- オープン/クローズ状態

## キャッシュ

GitHubデータはAPIコール削減のため30秒のTTLでキャッシュされます:

- PR一覧はリポジトリごとにキャッシュ
- issue一覧はリポジトリごとにキャッシュ
- CIチェックはチェックセクション展開時にオンデマンドで取得

## APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/github/prs` | オープンPR一覧（クエリ: `repo`） |
| GET | `/api/github/checks` | CIチェック一覧（クエリ: `repo`、`branch`） |
| GET | `/api/github/issues` | issue一覧（クエリ: `repo`） |
