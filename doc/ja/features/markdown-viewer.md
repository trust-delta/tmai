# Markdownビューア

tmaiダッシュボードからプロジェクトのドキュメントを直接閲覧・編集できます。

## 概要

Markdownビューアは、プロジェクト内のMarkdownファイルやテキストファイルのファイルツリーとプレビューパネルを提供します。初回読み込み時には `CLAUDE.md` を自動検出して選択します。

## アクセス方法

1. サイドバーでプロジェクトを選択
2. Markdown/ドキュメントオプションをクリックしてMarkdownパネルを開く

<!-- screenshot: markdown-viewer.png -->

## 機能

### ファイルツリー

左側のサイドバーに再帰的なファイルツリーを表示します:

- **フォルダ** -- クリックで展開/折りたたみ
- **ファイル** -- クリックで選択しプレビュー表示
- **拡張子バッジ** -- ファイルタイプラベル（`.md`、`.toml`、`.json` など）
- **読み取り専用インジケーター** -- ファイルが編集不可の場合に表示

### Markdownプレビュー

選択されたMarkdownファイルはGitHub Flavored Markdown (GFM) 対応でレンダリングされます:

- 見出し、リスト、テーブル
- シンタックスハイライト付きコードブロック
- リンクと画像
- タスクリスト（チェックボックス）

### 編集

対応する拡張子のファイルはその場で編集可能です:

1. **Edit** をクリックして編集モードに切り替え
2. テキストエリアで内容を修正
3. **Save** をクリックして変更を保存、または **Cancel** で破棄

#### 編集可能なファイルタイプ

| 拡張子 | タイプ |
|--------|--------|
| `.md` | Markdown |
| `.json` | JSON |
| `.toml` | TOML |
| `.txt` | プレーンテキスト |
| `.yaml` / `.yml` | YAML |

その他のファイルタイプは読み取り専用で表示されます。

## APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/files/md-tree` | ファイルツリーを取得（クエリ: `root`） |
| GET | `/api/files/read` | ファイル内容を読み取り（クエリ: `path`） |
| POST | `/api/files/write` | ファイル内容を書き込み |
