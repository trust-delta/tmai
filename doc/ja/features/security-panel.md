# セキュリティパネル

Claude Code設定とMCPサーバー設定のセキュリティ脆弱性をスキャンします。

## 概要

セキュリティパネルは設定ファイルの潜在的なセキュリティリスクを分析し、深刻度別に分類して、対処可能な詳細情報を表示します。

## アクセス方法

ステータスバーのセキュリティボタンをクリックしてセキュリティパネルを開きます。

<!-- screenshot: security-panel.png -->

## スキャンの実行

1. セキュリティパネルを開く
2. **Scan** をクリックしてセキュリティスキャンを実行
3. 深刻度別にグループ化された結果を確認

最後のスキャン結果はキャッシュされ、パネルを再度開いた時に自動的に表示されます。

## リスクカテゴリ

| カテゴリ | 説明 |
|----------|------|
| **Permissions** | Claude Code設定における過度に広範なファイルまたはコマンド権限 |
| **MCP Server** | セキュアでないMCPサーバー設定 |
| **Environment** | 機密性の高い環境変数やクレデンシャル |
| **Hooks** | 潜在的に危険なフック設定 |
| **FilePermissions** | ファイルレベルの権限の問題 |

## 深刻度レベル

| レベル | 色 | 説明 |
|--------|------|------|
| **Critical** | 赤 | 即座に対処が必要なセキュリティリスク |
| **High** | オレンジ | 重大な脆弱性 |
| **Medium** | 黄 | 対処が推奨される中程度のリスク |
| **Low** | 青 | 軽微な懸念事項またはベストプラクティスの提案 |

## スキャン結果

各検出項目には以下が含まれます:

- **Rule ID** -- トリガーされたセキュリティルールの識別子
- **Category** -- リスクが属するカテゴリ
- **Source** -- リスクが検出されたファイル
- **Description** -- 脆弱性の説明
- **Matched Value** -- 検出をトリガーした実際の設定値

## サマリービュー

パネル上部に深刻度レベルごとのカウントを含むサマリーが表示されます:

- スキャンされた総ファイル数
- スキャンされた総プロジェクト数
- スキャンのタイムスタンプ

## Exfil検出との関係

セキュリティパネルは設定ファイルの静的分析を行います。AIエージェントによる外部データ送信のランタイム検出については、[Exfil検出](./exfil-detection.md)を参照してください。

## APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/security/scan` | セキュリティスキャンを実行 |
| GET | `/api/security/last` | 最後のスキャン結果を取得 |
