# 使用量トラッキング

Claude Max/Proサブスクリプションのトークン消費量と支出を監視します。

## 概要

使用量パネルは、Claudeサブスクリプションのリアルタイム使用量メーターを表示し、レート制限の回避と支出の追跡に役立ちます。

## アクセス方法

使用量パネルはサイドバーの下部にあります。クリックして展開すると詳細情報が表示されます。

<!-- screenshot: usage-panel.png -->

## 機能

### 折りたたみ時の表示

折りたたみ時には、ヘッダー行にコンパクトなバッジとして上位2つの使用量メーターが更新ボタンとともに表示されます。

### 展開時の表示

ヘッダーをクリックして展開すると、すべてのメーターが表示されます:

- **プログレスバー** -- 色分けされた使用率の視覚的表示
- **メーター名** -- サブスクリプションティアと制限タイプ
- **リセット情報** -- 使用量カウンターのリセットタイミング
- **支出詳細** -- 該当する場合のドル金額
- **最終更新** -- データ取得時刻の相対タイムスタンプ

### 色分け

| 使用量レベル | 色 | 意味 |
|-------------|------|------|
| < 50% | シアン | 通常使用 |
| 50% - 80% | アンバー | 制限に近づいている |
| > 80% | 赤 | 制限付近または制限到達 |

### リアルタイム更新

使用量データはSSEイベントにより自動更新されます。サーバー側で使用量が変化すると、手動更新なしでパネルが更新されます。

### 手動更新

更新ボタン（↻）をクリックすると、プロバイダーからの使用量データ取得を手動でトリガーできます。

## 設定

`~/.config/tmai/config.toml`:

```toml
[usage]
enabled = true           # 使用量トラッキングを有効化（デフォルト: true）
auto_refresh_min = 5     # 自動更新間隔（分）
```

## APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/usage` | 現在の使用量データを取得 |
| POST | `/api/usage/fetch` | プロバイダーからの使用量取得をトリガー |
| GET | `/api/settings/usage` | 使用量トラッキング設定を取得 |
| PUT | `/api/settings/usage` | 使用量トラッキング設定を更新 |
