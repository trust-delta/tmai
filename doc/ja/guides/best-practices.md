# ベストプラクティス

tmaiを効果的に使うためのおすすめの方法。

## はじめに

### Claude Code Hooksをセットアップ（初回のみ）

`tmai init` を一度実行して、イベント駆動型の高精度状態検出のためにHTTP Hooksを有効化：

```bash
# 初回セットアップ
tmai init
```

これ以降、すべてのClaude Codeセッションが自動的にtmaiにイベントを送信します。ラッパー不要です。

### 必要に応じてPTYラッピングを使用

外部送信検知やAskUserQuestionの完全な解析が必要な場合：

```bash
# 追加機能が必要な場合
tmai wrap claude

# HooksとPTYラッピングは併用可能（ステータスはhooksが優先）
```

## WebUIのコツ

### プロジェクトを登録

プロジェクトディレクトリを登録して、ブランチグラフ、GitHub連携、ワークツリー管理を利用：

1. ステータスバーの設定ボタン（⚙）をクリック
2. プロジェクトパスを追加
3. サイドバーでプロジェクトを選択してブランチグラフを表示

### ブランチグラフを活用

ブランチグラフは全ブランチ、PR、CIステータスのビジュアル概要を提供：

- ブランチをクリックして差分サマリーとアクションを表示
- 「AI Merge」や「AI Create PR」で操作を委任
- ブランチ先端でCIステータスを直接確認

### 設定監査を実行

定期的にClaude Code設定の脆弱性をスキャン：

1. ステータスバーのセキュリティボタンをクリック
2. 重要度別に検出結果を確認
3. CriticalおよびHigh重要度の問題に迅速に対処

### 使用量を監視

レート制限を避けるためにサブスクリプション使用量を確認：

- サイドバー下部のUsage Panelでメーター状況を表示
- 色はシアン → アンバー（50%） → レッド（80%）で変化
- Refreshをクリックして最新データを取得

### エージェント間メッセージングを活用

エージェント間の連携が必要な場合、「Send To」機能を使って手動コピペなしにコンテキストを受け渡し。

## TUIのコツ（`--tmux`モード）

### tmaiは専用ペインで実行

```
┌───────────────────────┬───────────────────────┐
│                       │                       │
│    エージェントペイン  │      tmaiペイン       │
│                       │                       │
└───────────────────────┴───────────────────────┘
```

### ビューモードを適切に使用

| モード | 最適な用途 |
|--------|----------|
| Split | 1-3エージェントをプレビュー付きで監視 |
| List | 多数のエージェントを一度に監視 |
| Preview | 単一エージェントの出力に集中 |

`Tab`で切り替え。

## マルチエージェントのコツ

### 命名規則（tmuxモード）

わかりやすいセッション/ウィンドウ名を使用：

```bash
# わかりやすい命名
tmux new-session -s project-a
tmux new-window -n feature-auth
# エージェントは次のように表示: project-a:feature-auth
```

### ワークツリーワークフロー

WebUIでは、ブランチグラフからワークツリーをビジュアルに作成・管理。tmuxモードでは：

```bash
git worktree add ../project-feature-a feature/a
cd ../project-feature-a
tmai wrap claude
```

## Agent Teamsのコツ

### チーム進捗を監視

- **WebUI**: チームデータはサイドバーにリアルタイム更新とともに表示
- **TUI**: `T` で全チーム確認、`t` でタスクオーバーレイ

### スキャン間隔の調整

```toml
[teams]
scan_interval = 2  # より高頻度な更新（デフォルト: 5）
```

## セキュリティのコツ

### 監査ログを有効化

```bash
tmai --audit
```

監査データの確認：

```bash
tmai audit stats
tmai audit misdetections
```

### Webサーバーの公開を制限

- 信頼できるネットワークで使用
- トークンURLを公に共有しない
- トークンベース認証（自動）を使用

## トラブルシューティング

### エージェントが検出されない

1. `tmai init` が実行されたか確認（Claude Code hooks用）
2. tmuxモードの場合、エージェントがtmuxペインにあるか確認
3. プロセス名が対応エージェントと一致するか確認
4. `tmai --debug` でログを確認

### 状態が誤検出される

1. `tmai init` を実行してhooksを有効化（Claude Code推奨）
2. PTYラッピングで追加の精度向上（`tmai wrap claude`）
3. 検出ソースを確認：`◈ Hook` > `⊙ IPC` > `● capture`

### Webサーバーにアクセスできない

1. 設定で`[web] enabled = true`を確認
2. ファイアウォールがポートを許可しているか確認（デフォルト9876）
3. WSLの場合、ネットワーキングモードとポートフォワーディングを確認
