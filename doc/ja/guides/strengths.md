# tmaiの強み

tmaiが得意なこと、他のツールにはない特徴。

## 1. 統合WebUIダッシュボード

tmaiはAIエージェント管理のためのフル機能WebUIを提供 — ターミナル切り替え不要。

- **ブランチグラフ** — GitKrakenスタイルのレーンベースコミット可視化（PR/CIステータス付き）
- **GitHub連携** — PRレビュー状況、CIチェック、Issue追跡
- **ワークツリー管理** — ワークツリーの作成・削除・管理をビジュアルに
- **インタラクティブターミナル** — WebSocket I/O付きフルxterm.jsターミナル
- **セキュリティスキャン** — 設定の脆弱性を検出
- **使用量トラッキング** — サブスクリプションのトークン消費量を監視

コマンド一つで起動: `tmai` — Chrome App Modeが自動的に開きます。

## 2. 1パネル完結の操作

エージェントペインに**アタッチ不要**で承認操作 — WebUIとTUIの両モードで。

```
┌─────────────────────────────────────────────────────────────┐
│ 一般的なツール                                               │
│                                                             │
│  監視 → アタッチ → y入力 → デタッチ → 監視                 │
│         (3ステップ)                                         │
│                                                             │
│  問題: 操作中は他のエージェントが見えない                   │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ tmai                                                        │
│                                                             │
│  承認ボタンクリック（WebUI）またはyキー（TUI）               │
│  (1ステップ)                                                │
│                                                             │
│  利点: 全エージェントを見ながら即座に対応                   │
└─────────────────────────────────────────────────────────────┘
```

## 3. AskUserQuestion完全対応

Claude Codeの `AskUserQuestion` ツールに完全対応。

- 番号キー（1-9）またはクリック（WebUI）で直接選択
- Spaceキーでトグル（TUI）またはチェックボックス（WebUI）で複数選択
- 入力バーでのフリーテキスト入力

**他のツール**: 多くはy/n承認のみ、選択肢への対応なし。

## 4. 導入障壁の低さ

tmaiは**既存のワークフローを変えずに**導入できます。

- **WebUI（デフォルト）**: tmux不要。`tmai init && tmai` だけ
- **TUIモード**: 既存のtmux環境にtmaiを追加するだけ
- **Hooks**: 初回の `tmai init` — 以降は通常通り `claude` を使用
- エージェントの起動方法を変える必要なし

「とりあえず監視だけ」から始めて、段階的に機能を採用。

## 5. 3段構えの高精度検出

tmaiは最大の精度を実現するために3段構えの検出戦略を使用：

| 優先度 | 方式 | 精度 | 要件 |
|--------|------|------|------|
| 1 | **HTTP Hooks** | イベント駆動（最高） | `tmai init` + ウェブサーバー |
| 2 | **IPC Socket** | リアルタイム（高） | `tmai wrap` |
| 3 | **capture-pane** | ポーリング（中程度） | tmuxのみ |

自動フォールバックにより、セットアップに関係なく検出が動作します。

詳細: [Claude Code Hooks連携](../features/hooks.md) | [PTYラッピング](../features/pty-wrapping.md)

## 6. AI委任Git操作

ブランチグラフから複雑なGit操作をAIエージェントに委任：

- **AI Merge** — マージコンテキストとコンフリクト解決指示付きでエージェントを起動
- **AI Create PR** — 正しいベースブランチでPRを作成するエージェント

AIが操作を自律的に処理し、ダッシュボードから進捗を監視できます。

## 7. セキュリティ監視

2層のセキュリティ：

- **セキュリティパネル** — Claude Code設定とMCP設定の脆弱性を静的解析
- **外部送信検知** — 外部データ送信のランタイム監視（PTYラップモード）

詳細: [セキュリティパネル](../features/security-panel.md) | [外部送信検知](../features/exfil-detection.md)

## 8. モバイルリモートコントロール

QRコードスキャン（TUIモード）またはURLアクセス（任意のデバイス）でスマホから操作。

- エージェント一覧のリアルタイム更新
- 承認ボタン
- AskUserQuestion選択
- テキスト入力

詳細: [モバイルリモートコントロール](../features/web-remote.md)

## 9. Agent Teams可視化

Claude Code Agent Teamsのチーム構造とタスク進捗を監視。

- 全チームとメンバーを表示
- タスクステータス（pending, in progress, completed）を追跡
- SSE経由のリアルタイム更新

詳細: [Agent Teams](../features/agent-teams.md)

## まとめ

| 強み | 説明 |
|------|------|
| WebUIダッシュボード | ブランチグラフ、GitHub、ワークツリー付きフル機能Webインターフェース |
| 1パネル完結 | アタッチ不要で即座に操作 |
| AskUserQuestion | 番号キーで選択肢を直接選択 |
| 導入障壁の低さ | 既存ワークフローを変えずに導入 |
| 3段構え検出 | Hooks → IPC → capture-paneフォールバック |
| AI Git操作 | マージとPR作成を委任 |
| セキュリティ | 設定スキャン + ランタイム外部送信検知 |
| リモート操作 | スマホから承認可能 |
| Agent Teams | チーム構造・タスク進捗を可視化 |
