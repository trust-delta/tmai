# マルチエージェント監視

複数のAIエージェントを同時に監視・操作するワークフロー。

## tmaiの特徴：統合ダッシュボード

tmaiは複数エージェントを単一のビューから操作できます — ターミナル間の切り替え不要。

## WebUIモード（デフォルト）

### セットアップ

```bash
# 1. hooksをセットアップ（初回のみ）
tmai init

# 2. tmaiを起動
tmai

# 3. 別々のターミナルで複数エージェントを起動
claude              # ターミナル1: feature作業
cd ~/other && claude  # ターミナル2: 別プロジェクト
```

tmaiが全エージェントを自動検出し、サイドバーにプロジェクト別にグループ化して表示します。

### 操作

- **エージェントをクリック** — サイドバーで選択
- **承認** — 承認ボタンをクリック
- **選択肢選択** — AskUserQuestionの番号付き選択肢をクリック
- **テキスト送信** — 入力バーに入力
- **ターミナル** — 任意のエージェントのインタラクティブターミナルを開く
- **Kill** — エージェントを終了

### エージェント間メッセージング

あるエージェントから別のエージェントにテキストを送信：

1. 送信元エージェントを選択
2. 「Send To」パネルで送信先エージェントを選択
3. メッセージを入力して送信

関連タスクに取り組むエージェント間の連携が可能です。

### 新規エージェントの起動

ブラウザを離れずにWebUIから追加エージェントを起動：

1. スポーンダイアログまたはワークツリーのアクションボタンを使用
2. コマンドを選択（claude、codex、gemini等）
3. 新しいエージェントがサイドバーに自動的に表示

詳細は[エージェント起動](../features/agent-spawn.md)を参照。

## TUIモード（`--tmux`）

### セットアップ

```bash
# まずhooksをセットアップ
tmai init

# tmuxウィンドウでエージェントを起動
tmux new-window -n agent1 && claude
tmux new-window -n agent2 && cd ~/project-b && claude

# 別ウィンドウでtmaiを起動
tmux new-window -n monitor
tmai --tmux
```

### ナビゲートと操作

| キー | 動作 |
|------|------|
| `j` / `↓` | 次のエージェントへ |
| `k` / `↑` | 前のエージェントへ |
| `y` | 承認 |
| `1-9` | AskUserQuestionの選択肢を選択 |
| `Space` | 複数選択時のトグル |
| `p` | パススルーモード |

## オーケストレーターエージェントパターン（推奨）

最も強力なマルチエージェントパターンは**オーケストレーター + サブエージェント**モデルです。1つのClaude CodeエージェントがtmaiのMCPツールを使って他のエージェントをプログラマティックに管理します。

```
┌──────────────────────────────────────┐
│ Orchestrator（メインエージェント）⚙   │  ← MCPツールを使用
│  ├── Sub-agent 1（worktree）         │  ← dispatch_issue
│  ├── Sub-agent 2（worktree）         │  ← dispatch_issue
│  └── Sub-agent 3（worktree）         │  ← dispatch_issue
└──────────────────────────────────────┘
```

オーケストレーターは：
- `dispatch_issue` でIssueをサブエージェントにディスパッチ
- `list_agents` と `get_ci_status` で進捗を監視
- `send_prompt` で修正指示を送信
- PRをマージしてworktreeをクリーンアップ

WebUIでは、オーケストレーターはバッジ付きで視覚的に区別され、エージェントリストの最上部に固定されます。

詳細は[Issue駆動オーケストレーション](./issue-driven-orchestration.md)と[MCPサーバー](../features/mcp-server.md)を参照。

## Agent Teams連携

Claude CodeのAgent Teams機能（複数エージェントがプロジェクトで協調作業）を使用する場合、tmaiでチーム構造とタスク進捗を可視化できます。

### WebUI

チームデータはサイドバーにリアルタイムSSE更新とともに表示されます：
- チームメンバーのステータス変化
- タスク完了通知

### TUIキーバインド

| キー | 動作 |
|------|------|
| `t` | 選択中のチームメンバーのタスクオーバーレイを表示 |
| `T` | チーム一覧画面（全チームとメンバー） |

詳細は[Agent Teams](../features/agent-teams.md)を参照。

## メリット

1. **コンテキストスイッチなし** — 全体を見ながら操作
2. **迅速な対応** — 承認待ちに即座に気づいて対応
3. **状況把握が容易** — どのエージェントが何をしているか一目瞭然
4. **エージェント間メッセージング** — エージェント間の連携（WebUI）
